# Источники данных

Проект агрегирует данные из двух основных независимых источников для обеспечения максимального покрытия.

## 1. GLONASS-IAC FTP (Информационно-аналитический центр КВНО)
**URL:** `ftp.glonass-iac.ru`

Основной источник данных по группировкам ГЛОНАСС и GPS.
- **Путь:** `/MCC/PRODUCTS/ultra/` (или `final`/`rapid`). Файлы статусов находятся в `/MCC/STATUS/YYYY/`.
- **Формат имен:** `Stark_YYDDDHH.sp3` (или `.clk`).
- **Особенности:** Использует проприетарные расширения для метаданных (`.glo`, `.hlt`), которые парсятся для получения информации о состоянии спутников.

## 2. NASA CDDIS HTTPS
**URL:** `cddis.nasa.gov`

Мировой архив GNSS-данных. Используется для получения продуктов IGS (International GNSS Service).
- **Продукты:** `igu` (ultra-rapid), `igr` (rapid), `igs` (final).
- **Формат:** Long-name формат IGS3 (например, `IGS0OPSULT_20261160000_02D_30S_CLK.CLK.gz`).
- **Аутентификация:** Требует NASA Earthdata Login. Реализован автоматический обмен Basic Auth на Bearer Token для обхода проблем с TLS-редиректами на Windows.

## Поддерживаемые форматы файлов
- **SP3 (Standard Product 3):** Позиции спутников и поправки часов.
- **RINEX CLK:** Специализированный формат для поправок часов с высоким разрешением.
- **HLT/GLO:** Текстовые файлы статусов и метаданных ГЛОНАСС-ИАЦ.

### Спецификация формата .glo / .gps / .bds / .gal / .qzs (Паспорт ОГ)
Файлы содержат информацию о текущем и историческом составе орбитальных группировок различных систем (ГЛОНАСС, GPS, BeiDou, Galileo, QZSS).
- **Кодировка:** CP1251 (для поддержки кириллицы в типах КА).
- **Формат даты:** `DD.MM.YY` (российский стандарт).
- **Ключевые колонки:**
    1. Идентификатор (R01, G01, C01, E01, J01...) — используется для привязки данных.
    2. Слот.
    3. Номер по НКУ (заводской номер).
    5. Тип КА (ГЛОНАСС-М, GPS-IIF, BDS-3...).
    6-8. Дата оценки (Assessment date).
    9-11. Дата запуска (Launch date).
    12-14. Дата ввода в эксплуатацию (Commission date).

### Спецификация формата .hlt (История состояний)
Файлы содержат интервалы времени и коды здоровья спутников для всех поддерживаемых ГНСС (мульти-системный формат).
- **Формат строки:** `SatID StartDate StartTime EndDate EndTime StatusCode`
- **Коды статуса:**
    - `0`: Здоров (Healthy)
    - `1`: Не здоров (Unhealthy)
    - `3`: Техническое обслуживание (Maintenance)
- **Использование:** Позволяет сопоставлять математически найденные аномалии с официальными сервисными работами.

### Спецификация формата .oe (Орбитальные элементы)
Файлы содержат уточненные Кеплеровские элементы орбит с шагом в 1 час.
- **Путь на FTP:** `/MCC/STATUS/YYYY/` (например, `GLO_260419.oe`).
- **Ключевые извлекаемые параметры:**
    - `orbit_a`: Большая полуось (км).
    - `orbit_e`: Эксцентриситет.
    - `orbit_i`: Наклонение (градусы).
- **Использование:** Используется для анализа влияния физических характеристик орбиты на стабильность атомных стандартов частоты.
