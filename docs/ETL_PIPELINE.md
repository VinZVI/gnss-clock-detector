# ETL Пайплайн

Процесс извлечения, преобразования и загрузки данных (Extract, Transform, Load).

## Общая схема
1. **Оркестрация (`etl.py`):** Управляет очередностью загрузки слотов времени.
2. **Транспорт (`ftp_client.py`, `nasa_client.py`):** Скачивание файлов с учетом кэширования и повторных попыток.
3. **Парсинг (`parsers.py`):** Преобразование текстовых форматов в унифицированные списки словарей.
4. **Валидация и Дедупликация:** Проверка на дубликаты (Primary Key: `sat_id` + `epoch` + `product_type`) перед записью в БД.
5. **Аналитика:** Автоматический расчет ежедневных метрик (дрейф, шум) и сохранение в `analytics_cache`.

## Режимы запуска
- **Ручной:** Через CLI: `python -m gnss_clock.etl --source nasa --days 3`.
- **Фоновый:** Через POST-запрос к API (используется в UI).
- **Плановый:** Через GitHub Actions (Weekly Health Check) или системный cron.

## Очистка данных
При каждом запуске ETL выполняется процедура `cleanup`, которая удаляет данные старше 14 дней (настраивается через `GNSS_ETL_RETAIN_DAYS`) для предотвращения переполнения БД.
